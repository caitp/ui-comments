comments (directive in module ui.comments )

Description

This directive is replaced by the transcluded containerTemplate from commentsConfig.

The comments container produces an isolate scope, and injected into the isolate scope is the the following:

  • comments: Collection of comments, shared with the parent scope.

TODO: Expose the child-container status in status, so that it may be ng-if'd in templates.

The container should contain an ng-repeat directive for child comments. A very simple example of the containerTemplate might look like the following:

<div class="comments">
  <comment ng-repeat="comment in comments" comment-data="comment"></comment>
</div>

Usage

as element:
<comments
       comment-data="{expression}"
       order-by="{string}">
</comments>
as attribute
<div comments
     comment-data="{expression}"
     order-by="{string}">
   ...
</div>

Directive info

  • This directive creates new scope.

Parameters

  • comment-data – {expression} –

    Data model containing a collection of comments.

  • order-by – {string} –

    Override the default orderBy value.
